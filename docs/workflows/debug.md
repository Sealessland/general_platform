# 排查故障

当行为异常、验证失败，或后续 agent 反馈“没做完”时，走这个流程。

## 步骤

1. 用最小命令复现问题。
2. 记录准确命令、退出码和关键输出。
3. 识别问题表现在哪一层、根因来自哪一层。
4. 优先做契约级修复，而不是大面积重写。
5. 条件允许时，修完后补回归检查。
6. 如果 AI 参与定位、复现、修复或测试生成，记录 AI 草案、人工修正、实际验证和剩余风险。

## 需要收集的证据

- 复现问题的命令
- 输入、fixture、配置和环境假设
- 文档或测试中定义的期望行为
- 日志或命令输出中的实际行为

## 完成证据

- 问题已无法复现，或阻塞原因已明确记录。
- 稳定可测的行为已有回归检查。
- AI 参与过的排查过程已经记录到 `../../AI_WORKFLOW.md`、风险归档或 PR 描述中。
- 如果问题不在验证脚本本身，`bash scripts/validate-workspace.sh` 通过。
