# 交付前验证

在交付结构性改动、文档改动或 agent 路由改动前，执行这个流程。

## 命令

```bash
bash scripts/validate-workspace.sh
```

## 这个检查会验证什么

- 顶层关键入口文件是否存在
- 核心文档是否存在
- 工作流文档是否写明完成证据
- agent 路由文件是否指向规范文档
- `ci/` 目录下的 CI/CD 说明和基础脚本是否齐全

## 完成证据

- 验证命令退出码为 0
- 如果有刻意保留的缺口，已经在最近的相关文档中说明
- 本地提交信息已经基于暂存差异复核，能说明具体行为变化，未使用 `risk gaps`、`improve backend`、`update files` 这类空泛表述
