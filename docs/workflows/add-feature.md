# 新增或修改功能

当你要增加产品行为、接口、页面、任务入口或公共命令时，走这个流程。

## 步骤

1. 阅读 `../architecture.md`，先判断改动落在哪一层。
2. 先定义公共面：接口、页面路由、事件、命令或文档行为。
3. 可复用规则放到领域层，流程编排放到运行层，环境相关逻辑放到集成层。
4. 增加最小可执行验证，证明这个公共面真的成立。
5. 若工作流、契约或扩展点发生变化，同步更新文档。
6. 若 AI 影响了需求、方案、实现、测试或验收，同步记录到 `../../AI_WORKFLOW.md`、PR 描述或更具体的风险/ADR 文档。

## 不要默认假设

- 不要因为仓库空就随手选框架。
- 不要把供应商细节塞进领域层。
- 不要把只有文字说明的验收标准当成已完成。

## 完成证据

- 功能拥有清晰的公共面。
- 存在对应验证，或明确记录了测试缺口。
- AI 参与内容、人工修正和验证证据已经留档，或确认本次没有 AI 影响仓库产物。
- `bash scripts/validate-workspace.sh` 通过。
