# 端到端用例

## 消费者主链路

1. 打开笔记流
2. 查看笔记详情
3. 点击挂载商品
4. 选择 SKU 并加入购物车
5. 查看结算预览
6. 携带幂等键提交订单
7. 模拟支付
8. 查看订单详情
9. 在未支付时取消订单，或支付后发起退款

自动化覆盖：

- 内存 HTTP 测试覆盖笔记/商品进入、结算预览、幂等下单、支付、发货、完成、取消和退款入口。
- PostgreSQL-backed HTTP 测试覆盖真实数据库路径下的结算预览、幂等下单、库存预锁、支付确认、发货、完成、取消库存释放和退款库存恢复。
- 应用层测试覆盖从选中购物车项结算、未选中购物车项保留、非法 SKU、库存不足和跨商家 checkout 拒绝。

## 商家主链路

1. 创建商品
2. 创建 SKU
3. 上架或下架商品
4. 查看待发货订单并发货
5. 审批退款
6. 查看漏斗和汇总看板

自动化覆盖：

- HTTP 测试覆盖消费者访问商家接口被拒绝、商品创建/编辑、SKU 创建/编辑、上下架、发货非法状态、dashboard summary/funnel/products。
- 应用层测试覆盖商家所有权边界，其他商家不能修改商品、SKU 或履约订单。

## AI Copilot 链路

1. 生成商品卖点
2. 生成经营复盘
3. 查看 AI 任务记录中的输入、输出和状态

自动化覆盖：

- 后端 HTTP 测试覆盖商品卖点生成、经营复盘生成、消费者调用 AI 接口被拒绝、跨角色/跨商家读取 AI 任务返回 not found。
- 应用层测试覆盖 AI Provider 失败时任务状态落为 failed 并保存错误信息。
- AI service 单元测试覆盖 mock Provider 的商品卖点、经营复盘和非法输入。

## 当前非目标

- 还没有浏览器自动点击前端页面的 E2E 测试；前端自动化当前停留在源码守卫、类型检查、lint 和构建。
- 还没有外部压测工具对真实端口进行多连接压测；运行时 QPS 基线使用 PostgreSQL-backed Go benchmark。
