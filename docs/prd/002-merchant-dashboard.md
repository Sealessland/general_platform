# PRD 002：商家经营看板

## 背景

商家不只需要卖货，还需要围绕商品、库存、订单和转化数据做经营判断。

## 用户故事

作为商家，我希望能管理商品和 SKU，并查看经营漏斗和订单履约情况，以便优化转化和降低退款风险。

## MVP 范围

- 商品创建与编辑
- SKU 与库存管理
- 订单列表、订单详情、发货
- 退款审批
- 漏斗指标：浏览、点击、加购、下单、支付、退款
- 库存预警

## 验收标准

- [ ] 商家接口统一挂在 `/api/merchant` 下
- [ ] 看板接口返回漏斗和商品经营数据
- [ ] 库存更新路径具备可审计记录
- [ ] 每个新增接口都同步更新契约和测试
